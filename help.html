üöÄ Installation

Method 1: HACS (Recommended)

‚úì Easiest method - Automatic updates and version management
  1. Open HACS in Home Assistant
  2. Go to Frontend
  3. Click the ‚ãÆ menu and select Custom repositories
  4. Add repository: https://github.com/goatboynz/ha-grow-room-cards
  5. Category: Lovelace
  6. Click Install
  7. Restart Home Assistant
  8. Clear browser cache (Ctrl+Shift+R)

Method 2: Manual Installation

  1. Download the latest release
  2. Copy all .js files to config/www/grow-room-cards/
  3. Add resources in Home Assistant:
    • Go to Settings ‚Üí Dashboards ‚Üí Resources
    • Click Add Resource
    • URL: /local/grow-room-cards/grow-room-cards.js
    • Resource type: JavaScript Module
  4. Restart Home Assistant
  5. Clear browser cache (Ctrl+Shift+R)

Verify Installation

After installation, you should see the cards available when adding a new card to your dashboard. 
1. Edit your dashboard
2. Click "Add Card"
3. Search for "grow"
4. You should see all 12 grow room cards listed

üå∏ Room Overview Card

Complete dashboard showing all room metrics, day counter, sparklines, and alerts in one view.

Features

  • Room header with stage, day/week counter, and lights status
  • Last watering event tracker
  • System alerts with color coding
  • Environmental metrics with 24-hour sparkline graphs
  • Dark gradient theme matching professional grow room displays

Basic Configuration

type: custom:grow-room-overview-card
room_name: F2 Flowering Room
start_date_entity: input_datetime.f2_flower_start
lights_entity: light.f2_grow_lights
temperature_entity: sensor.f2_temperature
humidity_entity: sensor.f2_humidity
vpd_entity: sensor.f2_vpd
co2_entity: sensor.f2_co2

Full Configuration

type: custom:grow-room-overview-card
title: F2 Flowering Room
room_name: F2 Flowering Room
start_date_entity: input_datetime.f2_flower_start
stage_entity: input_select.f2_grow_stage
lights_entity: light.f2_grow_lights
temperature_entity: sensor.f2_temperature
humidity_entity: sensor.f2_humidity
vpd_entity: sensor.f2_vpd
leaf_vpd_entity: sensor.f2_leaf_vpd
co2_entity: sensor.f2_co2
last_watering_entity: sensor.f2_last_watering
alert_entities:
  - binary_sensor.f2_high_temp
  - binary_sensor.f2_low_humidity
show_sparklines: true
light_hours: 12

Required Entities Setup

Start Date Entity

input_datetime:
  f2_flower_start:
    name: F2 Flower Start Date
    has_date: true
    has_time: false

Stage Entity (Optional)

input_select:
  f2_grow_stage:
    name: F2 Grow Stage
    options:
      - Seedling
      - Vegetative
      - Flowering
      - Harvest
    initial: Flowering

Configuration Options

Option Type Required Description
room_name string Yes Name of the grow room
start_date_entity string Yes Entity tracking grow start date
lights_entity string Yes Light switch/entity
show_sparklines boolean No Show 24-hour trend graphs (default: true)

üîß Troubleshooting & FAQ

Cards Not Showing Up

Problem: Cards don't appear in the "Add Card" menu

Solutions:

  1. Verify resource is added correctly in Settings ‚Üí Dashboards ‚Üí Resources
  2. Check the URL path is correct: /local/grow-room-cards/...
  3. Clear browser cache (Ctrl+Shift+R or Cmd+Shift+R)
  4. Restart Home Assistant
  5. Check browser console for errors (F12)

"Custom element doesn't exist" Error

Problem: Error message when adding card

Solutions:

  1. Ensure the JavaScript file is in the correct location
  2. Verify the resource type is set to "JavaScript Module"
  3. Check file permissions (should be readable)
  4. Try manual installation if HACS isn't working

Entities Not Updating

Problem: Card shows "unavailable" or old data

Solutions:

  1. Verify entity IDs are correct in Developer Tools ‚Üí States
  2. Check that sensors are actually updating
  3. Ensure recorder is enabled for history features
  4. Check entity permissions and availability

Sparklines/Graphs Not Showing

Problem: History graphs or sparklines are blank

Solutions:

  1. Ensure entities have at least 24 hours of history
  2. Check recorder configuration includes these entities
  3. Verify history API is accessible
  4. Try disabling sparklines: show_sparklines: false

Sound Notifications Not Working

Note: Browser security requires user interaction before playing sounds

Solutions:

  1. Click anywhere on the page first
  2. Check browser sound permissions
  3. Verify enable_sound: true in config
  4. Test in different browser

Performance Issues

Problem: Dashboard is slow or laggy

Solutions:

  1. Disable sparklines on multiple cards
  2. Reduce history fetch intervals
  3. Use fewer cards per view
  4. Check Home Assistant system resources
  5. Clear browser cache and cookies

Common Configuration Mistakes

Issue Cause Fix
Card shows "Loading..." Missing required entity Check all required entities exist
Wrong data displayed Incorrect entity ID Verify entity IDs in Developer Tools
Tabs not working JavaScript error Check browser console (F12)
Styling looks wrong Theme conflict Try default theme or clear cache

üöÄ Quick Start Guide

Step 1: Install the Cards

Follow the installation instructions above using HACS or manual installation.

Step 2: Set Up Required Entities

Create the necessary input helpers and sensors:

# configuration.yaml
input_datetime:
  flower_start_date:
    name: Flower Start Date
    has_date: true
    has_time: false

input_select:
  grow_stage:
    name: Grow Stage
    options:
      - Seedling
      - Vegetative
      - Flowering
      - Harvest

Step 3: Add Your First Card

  1. Edit your dashboard
  2. Click "Add Card"
  3. Search for "grow"
  4. Select "Room Overview Card"
  5. Configure with your entities

Step 4: Customize

Add more cards and customize the layout to match your setup!

‚úì Pro Tip: Start with the Room Overview Card for a complete dashboard, then add specialized cards as needed.
